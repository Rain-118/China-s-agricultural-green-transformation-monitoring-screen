## **摘要**

 

农业绿色转型是当前中国农业可持续发展的重要方向。自2015年化肥使用量零增长行动实施以来，国家持续出台化肥农药减量政策，推动农业生产方式绿色化转型。为直观展示政策对农业绿色转型的驱动作用，本文设计并实现了一套基于Web技术的农业减量增效监测大屏系统。该系统以"政策驱动—数据响应—促进建议"为核心叙事逻辑，通过政策驱动轴串联五个关键政策节点，围绕投入减量、产出保障、替代机制、经济响应和效率提升五个维度构建了六大可视化模块。系统采用Node.js+Express构建后端API，MySQL数据库存储12,284条指标记录，利用Vue 3+TypeScript+Vite框架与ECharts 5图表库实现前端可视化，以全屏响应式布局适配多分辨率。系统支持31个省份2016-2024年化肥、粮食产量、机械动力等18类数据的动态展示，通过政策时间轴驱动六大模块数据联动，并构建了政策响应度指标体系和7项衍生计算指标，为政策制定者提供了从数据洞察到行动建议的完整决策支持链路。

 

**关键词**：农业绿色转型；化肥减量增效；数据可视化；政策响应度；监测大屏；ECharts

 

 

## **Abstract**

 

Agricultural green transformation is a critical direction for China's sustainable agricultural development. Since the implementation of the Zero-Growth Action Plan for Chemical Fertilizer Use in 2015, the Chinese government has continuously introduced policies to reduce chemical fertilizer and pesticide usage, driving the greening of agricultural production methods. To visually demonstrate the driving effect of policies on agricultural green transformation, this paper designs and implements a Web-based agricultural reduction and efficiency monitoring dashboard system. The system adopts "policy drive—data response—action recommendations" as its core narrative logic, connecting five key policy milestones through a policy-driven axis, and constructs six visualization modules across five dimensions: input reduction, output guarantee, substitution mechanisms, economic response, and efficiency improvement. The system employs Node.js+Express for the backend API with MySQL database storing 12,284 indicator records, and utilizes Vue 3+TypeScript+Vite framework with ECharts 5 for frontend visualization, delivering full-screen responsive layout across multiple resolutions. It supports dynamic display of 18 categories of data across 31 provinces from 2016 to 2024, featuring a policy timeline that drives synchronized data updates across six dashboard modules. A policy responsiveness indicator system with seven derived calculation indicators was constructed, providing policymakers with a complete decision-support chain from data insight to actionable recommendations.

 

**Keywords**: agricultural green transformation; chemical fertilizer reduction and efficiency; data visualization; policy responsiveness; monitoring dashboard; ECharts

 

 

## **1 绪论**

 

### **1.1 研究背景与研究意义**

 

农业是中国国民经济的基础产业，也是保障国家粮食安全的关键领域。长期以来，中国农业生产高度依赖化肥、农药等化学投入品，化肥施用强度长期高于国际平均水平。2015年，中国农用化肥施用折纯量达到6022.6万吨的历史峰值，化肥过量施用导致的农业面源污染、土壤板结、水体富营养化等问题日益突出[1]。

 

为应对这一挑战，国家层面持续出台化肥减量政策：2015年原农业部发布《到2020年化肥使用量零增长行动方案》，明确提出化肥使用量零增长目标；2017年农业部启动农业绿色发展五大行动，从单项突破转向系统推进；2021年农业农村部等六部门联合印发《"十四五"全国农业绿色发展规划》，首次将农业绿色发展纳入国家五年规划并将化肥利用率纳入政绩考核；2022年农业农村部发布《到2025年化肥减量化行动方案》，设定了量化减量目标；2024年中央一号文件明确提出"推进化肥农药减量增效"，将减量增效提升至国家最高文件层面[2-4]。

 

在"政策密集出台—数据持续变化"的背景下，各级政策制定者面临以下核心问题：政策是否真正推动了农业绿色转型？哪些省份积极响应了政策？哪些省份需要进一步推动？减量是否影响了粮食安全和农民收入？下一阶段政策应在哪里发力？这些问题需要一个能够将政策链条与数据变化进行关联分析的可视化工具来回答[5]。

 

传统的农业数据展示方式以静态报表和简单图表为主，难以揭示政策驱动力与数据变化之间的因果关系，也无法支撑"从数据到行动"的决策链路。因此，设计一套以"政策驱动"为核心叙事逻辑的监测大屏系统，具有重要的实践价值和示范意义。

 

### **1.2 国内外研究现状**

 

在数据可视化领域，Stephen Few提出了仪表盘设计的核心原则，强调可视化应以支持决策为目标，而非单纯展示数据[6]。Edward Tufte在《量化信息的视觉呈现》中提出了"数据墨水比"最大化原则，主张简化非必要的视觉元素[7]。这些设计理念对本系统的视觉设计具有重要指导意义。

 

在农业数据可视化方面，国际研究主要集中在精准农业和智慧农业领域。联合国粮农组织（FAO）开发了FAOSTAT数据平台，提供全球农业数据的可视化查询和分析功能。美国农业部（USDA）建立了国家农业统计服务（NASS）数据平台，通过交互式地图和图表展示各州农业数据。

 

在国内，随着大数据技术的发展和数字政府建设的推进，政务数据大屏在农业领域逐渐兴起。农业农村部建立了全国农业信息监测预警平台，部分省份也开发了农业大数据分析平台。然而，现有平台多侧重于数据展示和统计报表，较少将政策维度纳入可视化设计，缺乏"政策—数据—行动"的闭环分析框架[8]。

 

在技术选型方面，ECharts作为Apache旗下的开源可视化库，具有丰富的图表类型和良好的地图支持，已被广泛应用于政务大屏场景[9]。Vue 3框架以其渐进式设计、优秀的响应式数据绑定和组件化能力，成为前端开发的优先选择之一。

 

### **1.3 研究内容**

 

本文围绕农业减量增效政策响应监测大屏的设计与实现，开展以下研究工作：

 

\1. **政策响应度指标体系构建**：基于化肥减量率、粮食增产率、效率变化率等指标，构建多维度的政策响应度评价体系。

\2. **可视化大屏方案设计**：以"政策驱动轴"为核心视觉元素，设计投入减量、产出保障、替代机制、经济响应、效率提升和促进建议六大功能模块。

\3. **数据处理与分析**：针对18类原始数据和7项衍生指标，设计Python数据预处理流程。

\4. **系统设计与实现**：基于Vue 3和ECharts实现单页面全屏自适应大屏，支持31省份数据的动态展示与交互。

\5. **系统测试与验证**：对系统功能、性能和兼容性进行全面测试。

 

 

## **2 相关理论与技术**

 

### **2.1 数据可视化技术**

 

数据可视化是将抽象数据转化为可视化图形表示的技术，旨在增强人类对数据模式、趋势和异常的理解能力。本系统采用了以下可视化图表类型：

 

\1. **地图热力图**：用于展示31省份政策响应度的空间分布，通过颜色深浅直观反映响应度高低。

\2. **双轴折线图**：用于同时展示化肥使用量和粮食产量的变化趋势，直观呈现"一降一升"的反向关系。

\3. **四象限散点图**：以化肥减量率为横轴、粮食增产率为纵轴，将31省份分为四个象限，快速识别"减量多+增产多"的榜样省份和"减量多+减产"的危险省份。

\4. **堆叠柱状图**：展示化肥结构演变，N/P/K段逐年缩窄与复合肥段逐年增宽的对比，体现施肥结构从粗放走向精准。

\5. **散点气泡图**：横轴为减量率、纵轴为产值变化率、气泡大小为收入涨幅，实现三维信息的同时呈现。

 

### **2.2 Node.js数据处理与MySQL数据库技术**

 

系统采用Node.js进行数据导入和处理，MySQL作为持久化存储。使用的核心技术包括：

 

\1. **csv-parse库**：用于解析国家统计局分省年度数据CSV文件，支持A类文件（单文件横表含31省）和B类文件（每省独立纵表）两种格式。

\2. **mysql2库**：提供MySQL数据库连接池，支持Promise异步查询，实现高效数据存取。

\3. **Express.js框架**：构建RESTful API服务，提供`/api/dashboard`等端点，在服务端完成数据聚合与衍生指标计算。

 

### **2.3 Web前端技术**

 

系统前端采用Vue 3框架和ECharts图表库，技术选型理由如下：

 

\1. **Vue 3 + TypeScript**：采用Composition API和`<script setup>`语法糖，代码组织清晰；响应式数据绑定使数据更新自动触发视图刷新；配合Vite构建工具实现模块化开发和HMR热更新。

\2. **ECharts 5.5**：Apache旗下成熟的开源可视化库，内置中国地图支持，图表类型丰富（地图、折线、柱状、面积、雷达等），交互功能完善。

\3. **阿里云DataV GeoJSON**：提供中国省级行政区划GeoJSON数据，CDN动态加载用于地图渲染。

\4. **CSS Grid + Flexbox布局**：实现8行×3列响应式网格布局，两侧30%弹性列+中间全屏地图背景层，配合vh/vw单位实现全屏自适应。

## **4 数据获取及分析**

 

### **4.1 数据来源**

 

系统数据全部来源于国家统计局公开的分省年度数据，涵盖2016-2024年31个省（自治区、直辖市）的17类原始指标，具体包括：

 

**A类（化学投入品，7项）**：农用化肥施用折纯量（总量）、氮肥施用折纯量、磷肥施用折纯量、钾肥施用折纯量、复合肥施用折纯量、农药使用量、农用塑料薄膜使用量。

 

**B类（产出数据，2项）**：粮食产量（含夏收/秋粮/谷物/稻谷/小麦/玉米/豆类/薯类分项）、农作物播种面积（含粮食作物/夏收/秋收/谷物分项）。

 

**C类（基础设施，3项）**：农业机械总动力（含大中型/小型拖拉机等分项）、有效灌溉面积、农村水电发电量。

 

**D类（经济民生，4项）**：农林牧渔业总产值（含农/林/牧/渔分项）、第一产业增加值与GDP、农村居民人均可支配收入、农村居民人均消费支出。

 

**E类（人口背景，2项）**：乡村人口、城镇人口。

 

### **4.2 数据预处理**

 

数据原始格式分为两类，需分别处理：

 

**A类文件**（单文件横表格式）：文件第一行为数据库名称，第二行为时间范围，第三行为表头（地区+各年份），后续每行一个省份数据。数据以Tab分隔，年份值以逗号包裹。解析时需strip逗号后转换为float，缺失值（,）则记录为null。

 

**B类文件**（每省纵表格式）：每省一个独立CSV文件。第一行为数据库名称，第二行为时间信息，第三行为表头（指标+各年份），后续每行一个指标值。解析时需先按文件名匹配省份，再按指标名称匹配数据行。

 

数据预处理流程如下：

 

\1. 遍历A类CSV文件，解析为"省份×年份"矩阵。

\2. 遍历B类文件目录，读取每省CSV文件，解析为"指标×年份"矩阵，合并为统一的"省份×指标×年份"三维结构。

\3. 通过Node.js脚本（import_data.js）将全部数据导入MySQL数据库的indicator_data表（EAV结构：province, year, category, indicator, value），共计12,284条记录。

\4. Express API在服务端完成7项衍生指标计算和3维度政策响应度计算。

\5. API以JSON格式返回聚合数据供前端消费。

 

### **4.3 数据窗口说明**

 

大屏数据窗口为2016年至2024年共计9年，选择理由为：2015年"化肥使用量零增长行动方案"发布后，2016年作为政策效果观察起点，涵盖政策启动期（2016-2020）和加速期（2021-2024），形成完整的政策响应时间序列。2016年为基线年（各省与自身比较响应度为零），2024年为最新完整年份。2025年数据在采集时多为空值故未纳入。

### **4.5 政策响应度计算**

 

| **序号** | **指标名称**     | **计算公式**                           | **单位** |
| -------- | ---------------- | -------------------------------------- | -------- |
| ------   | ---------        | ---------                              | ------   |
| D1       | 化肥施用强度     | 化肥总量(万吨)×1000 ÷ 播种面积(千公顷) | kg/公顷  |
| D2       | 粮食单产         | 粮食产量(万吨)×1000 ÷ 播种面积(千公顷) | kg/公顷  |
| D3       | 单位化肥产粮比   | 粮食产量(万吨) ÷ 化肥总量(万吨)        | 吨/吨    |
| D4       | 机械化密度       | 机械总动力(万千瓦) ÷ 播种面积(千公顷)  | kW/公顷  |
| D5       | 氮肥占比         | 氮肥 ÷ 化肥总量 × 100                  | %        |
| D6       | 城镇化率         | 城镇人口 ÷ (城镇+乡村人口) × 100       | %        |
| D7       | 省际变异系数(CV) | 各省指标标准差 ÷ 均值 × 100            | %        |

 

政策响应度是本系统的核心分析指标，定义为：

 

政策响应度 = 该省化肥减量率 ÷ 全国平均减量率

 

 

分级标准：

\- ≥1.5：高度响应（绿色），政策驱动效果显著

\- 1.0-1.5：正常响应（蓝色），政策驱动有效

\- 0.5-1.0：弱响应（橙色），需加大政策推动力度

\- <0.5：未响应（红色），需重点督导

 

同一逻辑扩展至粮食增产响应度和效率提升响应度，形成多维度政策响应度画像。

 

 

## **5 系统设计与实现**

 

### **5.1 系统总体架构**

 

系统采用前后端分离的三层架构：

 

**数据层**：MySQL数据库（agriculture_green），EAV单表设计（indicator_data），存储31省+全国12,284条记录，覆盖17类原始指标×9年。

**服务层**：Node.js+Express API服务器（端口3001），提供`/api/dashboard`（主仪表盘聚合数据+衍生指标计算）、`/api/national-economy`（全国亩均数据）等RESTful端点。

**展示层**：Vue 3+TypeScript+Vite前端（端口5173），7个Vue组件（PolicyTimeline + 6个功能模块），通过Axios请求API数据，ECharts 5渲染图表。

 

CSV源文件 → Node.js导入脚本 → MySQL数据库 → Express API → JSON → Vue3前端 → 浏览器展示

 

 

### **5.2 系统功能模块设计**

 

系统以"政策驱动轴"为顶部通栏核心视觉元素，下方以模块1全屏地图为底层（z-index:1），模块2-5分列两侧（30%宽度），模块6嵌入地图底栏的布局方案：

 

**政策驱动轴**（顶部通栏）：横向时间轴展示2015零增长→2017五大行动→2021绿色规划→2022减量化方案→2024一号文件五个关键政策节点，可点击切换年份，驱动全部模块数据联动更新。2015年标记为"无数据"（数据从2016年开始）。

 

**模块1：投入减量响应度**（全屏底层）：中国省级地图全透明背景，滚轮缩放+拖拽平移，冲击波动画（以北京为原点声呐波纹扩散逐省点亮）。底部四级图例可点击筛选省份。响应度 = 省化肥减量率 ÷ 全国减量率（2016基线）。

 

**模块2：产出保障响应度**（右侧上）：双轴折线图展示全国化肥总量(↓红色)与粮食产量(↑绿色)的反向趋势，含markPoint极值标注，按时间轴选中年过滤数据。

 

**模块3：替代机制响应度**（右侧下）：堆叠面积图展示N/P/K/复合肥结构演变（左Y轴），机械总动力折线（右Y轴）置于面积层上方。内置ECharts图例可toggle系列显隐。

 

**模块4：经济响应度**（左侧下）：双轴分组柱状图展示全国亩均施肥量vs亩均产值变化趋势，数据来自独立API端点（`/api/national-economy`）。

 

**模块5：效率响应度**（左侧上）：七大区域六维雷达图（肥料利用率、养分留存率、节水效率、面源污染降幅、土壤酸化改善、板结修复），自定义图例点击切换区域显隐。

 

**模块6：促进建议**（地图底栏浮层）：31省响应度色块网格（四色16列），与模块1图例和时间轴联动，省份字体为SimHei黑体。

import { createReadStream } from 'fs';

import { parse } from 'csv-parse';

import pool from './db.js';

 

async function importProvinceData(category, indicator, filePath) {

  const records = [];

  const parser = createReadStream(filePath).pipe(parse({ delimiter: '\t' }));

  // 解析CSV并构建EAV记录

  for await (const row of parser) {

    const [province, ...yearValues] = row;

    years.forEach((year, i) => {

      if (yearValues[i]) {

        records.push([province, year, category, indicator, parseFloat(yearValues[i])]);

      }

    });

  }

  // 批量插入MySQL

  await pool.query(

    'INSERT INTO indicator_data (province, year, category, indicator, value) VALUES ?',

    [records]

  );

}

## **7 总结与展望**

 

### **7.1 项目总结**

 

本文设计并实现了一套基于数据可视化的中国农业化肥减量增效政策响应监测大屏系统。系统以"政策驱动—数据响应—促进建议"为核心叙事逻辑，在以下方面取得了创新：

 

\1. **首创政策响应度指标体系**：将政策维度首次纳入农业数据可视化核心框架，提出了政策响应度的量化计算公式和四级分级标准，实现了从"数据展示"到"政策响应评估"的范式转变。

 

\2. **构建六大维度分析框架**：从投入减量、产出保障、替代机制、经济响应、效率提升五个维度加促进建议模块，形成了覆盖绿色转型全链条的分析体系。

 

\3. **构建前后端分离的数据驱动架构**：采用Node.js+Express+MySQL+Vue 3+TypeScript+Vite技术栈，实现了CSV→数据库→API→前端的数据管道，支持12,284条记录的存储与查询，前端通过Axios获取API数据实现图表渲染。

 

\4. **设计了"数据—洞察—行动"决策闭环**：不满足于展示"化肥降了多少"，而是回答"政策推动了多少""哪些省份推得动""下一步该往哪推"等行动导向问题。

 

### **7.2 不足与展望**

 

系统仍存在以下不足：

 

\1. **数据时效性限制**：数据依赖国家统计局年度发布，无法实现实时或高频数据更新。未来可考虑接入农业农村部监测点数据，缩短数据更新周期。

 

\2. **政策响应度模型可深化**：当前响应度计算基于单一指标比值，未能控制干扰变量（如产业结构差异、气候因素等）。后续可引入计量经济学模型进行更精确的因果效应估计。

 

\3. **移动端适配缺失**：系统当前仅适配桌面端分辨率，采用弹性网格布局（30%-40%-30%）适配不同屏幕宽度，但未针对平板和手机端进行专门适配。

 

\4. **交互深度可增强**：当前已实现时间轴联动、冲击波动画、图例筛选、滚轮缩放、卡片悬浮放大、雷达图区域切换等交互，未来可增加数据下钻、多省对比等更高级交互功能。

 

\5. **预测功能缺失**：系统仅展示历史数据，未来可引入机器学习模型，对化肥减量趋势和政策效果进行预测。
